# powerups-vaultbot3000

Automated journaling and reflection for [Claude Code](https://docs.anthropic.com/en/docs/claude-code) sessions. Every session gets a structured journal entry in your Obsidian vault. Thread documents and weekly digests synthesize patterns across sessions.

## Why

You run dozens of Claude Code sessions a week across multiple projects. Without a record, context evaporates between sessions — decisions get re-litigated, dead ends get re-explored, improvement ideas get lost. Manual notes don't happen.

This plugin writes a journal entry automatically when each session ends, then gives you tools to synthesize those entries into longitudinal thread documents and weekly digests. The journals live in Obsidian where you can search, link, and query them.

## What's In the Box

| Component | Purpose |
|-----------|---------|
| `skills/journal/` | Write a session journal — triggered automatically by session-end hook or manually via `powerups-vaultbot3000:journal` |
| `skills/reflect/` | Synthesize session journals into thread documents or weekly digests (`powerups-vaultbot3000:reflect`) |
| `hooks/hooks.json` | Plugin hook config — fires `journal-hook.sh` on SessionEnd |
| `scripts/journal-hook.sh` | SessionEnd hook — launches a background Claude process to journal the ended session |
| `scripts/extract-transcript.sh` | Standalone transcript extractor (jq pipeline also inlined in the journal skill) |
| `scripts/install-cron.sh` | Installs weekly digest cron job (Sunday 5am) |

## How It Works

```
Session ends → hook fires → background Claude runs /journal → Session journal written to Obsidian
```

The hook launches a new `claude --print` process that reads the just-ended session's transcript, summarizes it, and writes a structured journal entry via `notesmd` CLI. A recursion guard (`AGENT_JOURNAL_SESSION=1` env var) prevents the journal-writing session from journaling itself.

Thread documents and weekly digests are produced on demand:

- `/reflect` — reads all session journals, groups by thread, creates or updates thread documents
- `/reflect weekly` — generates a weekly digest with thread progress, recurring patterns, and improvement proposals

## Obsidian Vault Structure

```
~/obsidian/Agent Journals/
  Sessions/     One file per Claude Code session (auto-generated)
  Threads/      One file per line of work (updated by /reflect)
  Weekly/       One file per week (generated by /reflect weekly)
```

Session journals have YAML frontmatter with session ID, date, project path, thread names, and tags — all queryable via Obsidian Dataview. Wikilinks connect sessions to threads in the graph view.

## Platform

macOS and Linux only. The session-end hook, cron-based weekly digest, and shell scripts all assume a Unix environment. Windows (including WSL) is untested and unsupported.

## Prerequisites

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code)
- `notesmd` CLI — writes to Obsidian vault
- `jq` — transcript extraction
- `cron` — weekly digest scheduling
- An Obsidian vault with an `Agent Journals` folder

## Install

### Via Powerups Marketplace (recommended)

```
/plugin marketplace add njt/powerups-marketplace
/plugin install powerups-vaultbot3000@powerups-marketplace
```

Then set your vault path in `~/.claude/settings.json`:

```json
{
  "env": {
    "OBSIDIAN_VAULT": "~/obsidian"
  }
}
```

Set `OBSIDIAN_VAULT` to wherever your vault lives. Skills default to `~/obsidian` if unset.

Create the vault folders and install the weekly digest cron job:

```bash
mkdir -p ~/obsidian/"Agent Journals"/{Sessions,Threads,Weekly}
scripts/install-cron.sh
```

### Manual

```bash
git clone https://github.com/njt/powerups-vaultbot3000.git
cd powerups-vaultbot3000

# Skills
cp -r skills/journal ~/.claude/skills/
cp -r skills/reflect ~/.claude/skills/

# Scripts
mkdir -p ~/.claude/scripts
cp scripts/journal-hook.sh ~/.claude/scripts/
chmod +x ~/.claude/scripts/journal-hook.sh

# Create vault folders (adjust path to your vault)
mkdir -p ~/obsidian/"Agent Journals"/{Sessions,Threads,Weekly}

# Install weekly digest cron job (Sunday 5am)
./scripts/install-cron.sh
```

Add the session-end hook and vault path to `~/.claude/settings.json`:

```json
{
  "env": {
    "OBSIDIAN_VAULT": "~/obsidian"
  },
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude/scripts/journal-hook.sh",
            "timeout": 10
          }
        ]
      }
    ]
  }
}
```

## Usage

**Automatic journaling** — just use Claude Code normally. When a session ends, a journal entry appears in your vault's `Agent Journals/Sessions/`.

**Manual journaling** — type `/powerups-vaultbot3000:journal` in any session. Optionally pass a session JSONL path.

**Thread synthesis** — type `/powerups-vaultbot3000:reflect` to update thread documents based on all session journals.

**Weekly digest** — runs automatically via cron every Sunday at 5am. Can also be triggered manually with `/powerups-vaultbot3000:reflect weekly`.

## This Is Opinionated

- **Every session gets a journal** — unless the skill decides the session was too trivial (a test ping, an accidental start). The de minimis check is semantic, not a line count.
- **Threads are named by intent**, not by directory path. "Meeting Transcription Pipeline" not "/Users/gnat/Source".
- **Voice matters.** All generated prose uses the `nat-write` skill — no AI summary voice, no corporate boilerplate. Direct, concrete, honest about mess.
- **Journals are structured but not rigid.** YAML frontmatter for machine queries, markdown body for human reading. Sections can be short or skipped when there's nothing to say.
